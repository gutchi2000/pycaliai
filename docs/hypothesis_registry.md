## 仮説 H1：field_spread × 市場歪み

**タイトル:** 混戦レースにおける中位馬の市場過小評価

### 予測

field_spread を以下のように定義する：

```
field_spread = std(同一レース内スピード指数) / 出走頭数
```

field_spread が上位 25% のレース（高混戦群）において、
モデルがレース内 3〜5 番手に予測した馬の

**実際の複勝率 > 市場内包複勝率（= 1/複勝オッズ × 1.25）**

という関係が成立する。
かつ、この関係は低混戦群（下位 25%）では成立しない。

### 検証指標

```
高混戦群 3〜5 番手予測馬のROI > 低混戦群 3〜5 番手予測馬のROI
かつ 高混戦群のROI > 1.0
```

### 理論的根拠

多頭数・実力拮抗レースでは市場参加者の認知負荷が増加し、
中位馬への評価が粗くなる（認知負荷理論）。
モデルは全馬を均等に処理するため、この条件下で相対的優位を持つ。

「混戦 = どの馬も複勝に入りやすい」という単純な事実ではなく、
「混戦で市場が穴馬を系統的に過小評価する」という歪みを予測する。

### 棄却条件

- 高混戦群 ROI < 低混戦群 ROI
- または 高混戦群 ROI < 0.90（明確なマイナス）

## 仮説 H2：class_gap × 降級効果

**タイトル:** 1段降級における市場の過小評価

### 予測

class_gap を以下のように定義する：

```
class_gap = 前走クラス番号 - 現クラス番号
（昇級 = 負値、同クラス = 0、1段降級 = 1、2段降級 = 2）
```

**「1段降級（class_gap=1）の馬のROIは、
　 同クラス出走（class_gap=0）の馬のROIを上回る」**

2段降級（class_gap=2）の挙動については事前に断言しない。
（逆U字型・単調増加・フラットのいずれも理論的に排除できないため）

### 検証指標

```
1段降級ROI - 同クラスROI > +5%
（複勝ROI、◎予測馬対象、2024年7〜12月）
```

### 理論的根拠

直近不振の記憶が市場参加者の評価を歪める（recency bias）。
1段降級馬は前クラスで戦えた実力を持ちながら、
直近の負けだけを見て過小評価される。
この構造的な歪みは「1段降級 > 同クラス」として現れる。

なお、逆U字型仮説（2段降級 < 1段降級）を当初検討したが、
「市場が2段降級を察知して補正する」という推論は
recency bias の機序から演繹できず、憶測にとどまると判断。
仮説を「1段降級 > 同クラス」の1点に絞ることで棄却条件を明確化した。

### 棄却条件

```
1段降級ROI <= 同クラスROI
（差がゼロ以下なら、class_gapの追加に理論的根拠がない）
```

### 特徴量設計への含意

2段降級の形状を断言しない以上、class_gap は **連続値** として実装する。
LGBMが形状（逆U字型・単調・閾値型）を自動発見する。
実装後に部分依存プロット（PDP）で実際の形状を確認し、
H2の棄却判断とは独立して特徴量採用を評価する。

**廃止モデル:** `stacking_meta_v1.pkl` / `stacking_calibrator_v1.pkl`

**廃止理由:**

| 指標 | 値 | 閾値 | 判定 |
|------|-----|------|------|
| valid BrierScore改善 | 0.0010以下 | ≥0.002 | 実用価値なし |

- IsotonicRegressionの出力が全サンプルで max > 0.50 となり、
  スタッキング補正後も元のアンサンブル確率にフォールバックし続けていた

- `predict_weekly.py` にて `ensemble_predict()` 冒頭でコメントアウト済み

```
6モデル加重平均
LGBM×0.25 + CatBoost×0.25 + Rank×0.20 + Win×0.20 + FukuLGBM×0.05 + FukuCat×0.05
キャリブレーター: ensemble_calibrator_v4.pkl
```

**再学習・復活条件:**

### 重×16頭除外ルール（2026-03-25 削除）

**元のルール:**
```
baba == "重" and n_horses >= 16 → スキップ（ROI=50.5%）
```

**設定根拠（当時）:** valid 2023 単年 n=130行 ROI=55.2%

**廃止根拠:**

| 年 | n | ROI |
|----|---|-----|
| 2023（ルール根拠） | 130行 | 55.2% |
| 2024 | 142行 | 97.4% |
| 2025 | 104行 | 65.7% |
| **3年合計** | **376行** | **73.1%** |

- n=130（単年）で作成したルールを n=376 の3年観察が否定
- 3年合計ROI 73.1%は「除外すべき地獄セグメント（<60%）」の閾値を超えている
- 2024の逆転（97.4%）は n=142 で無視できる規模ではない
- 元ルールのサンプルが過少であり、ルール自体の信頼度が不十分だったと判断

**再検証条件:** n=500以上蓄積時（現在n=376）に同条件での3年以上のROI推移を確認する。
再検証時の判断基準: ROI<65%（3年平均）かつ n≥500 → 復活を検討。

**ただし本観察は今日のデータを見た後の発見であり、閾値調整の根拠には使用しない。**

**仮説H3:「EV除外の問題はEV3.0-4.0帯であり、EV4.0以上は除外不要」**

- 理論的根拠（事前）: EV4.0+は市場オッズとモデル確率の乖離が最大で、
  モデルが正しければ高リターンが期待できる。
  EV3.0-4.0は「過大評価」と「真の好機」が混在する中途半端な帯の可能性がある。
- **検証条件: EV3.0-4.0 および EV4.0+ それぞれ n≥500 蓄積後**
- 事前決定ルール:
  - EV4.0+ ROI ≥ 85% かつ EV3.0-4.0 ROI < 75% → 閾値を4.0に変更
  - 上記以外 → 現行閾値3.0を維持
- 現在のEV4.0+: n=664行（2023-2024）→ 2025蓄積でn=500超を確認してから検証

## strategy_weights.json 構造的問題記録（2026-03-25 発見）

**発見の経緯:**
今日の議論で「予測ターゲット」「損失関数」「特徴量欠落」より深刻な問題として浮上。
重×16頭ルール（n=130問題）の調査中に同一構造の問題を発見。

**問題の構造:**

```
採用判断: ROI_test >= 80%（test = 2024-2025データ）
評価:     同じ2024-2025データでROIを測定
↓
30エントリ全てが「testで良かったから採用 → testで測ると良い」の循環
```

**定量的証拠:**

| 場所 | クラス | 馬券種 | valid ROI | test ROI | 乖離 |
|------|--------|--------|-----------|----------|------|
| 函館 | 2勝 | 馬連 | 224.7% | 136.9% | **87.8pt** |
| 新潟 | 1勝 | 馬連 | 62.4% | 150.0% | **87.6pt** |
| 中京 | 2勝 | 単勝 | 128.7% | 81.6% | 47.1pt |

- 採用基準: `ROI_valid >= 50%`（旧モデル期間として緩設定）/ `ROI_test >= 80%`
- MIN_RACES = 15（各期間15レース以上）
- **サンプル数フィールドがJSONに存在しなかった** → 本日修正（n_races_valid/n_races_test追加）

**影響:**
- test期間（2024-2025）のROIは構造的に過大評価されている
- strategy_weights.jsonで「良さそうな条件」を選ぶほど、testROIが良く見える
- 実運用での実績が「バックテストより悪い」場合の主因になりうる

**修正方針:**
walk-forwardによる再設計が必要（別イテレーション）。
具体的には「valid→採用判断、test→独立評価」の分離、
または「2013-2022訓練、2023検証、2024-2025評価」の3分割設計。

**暫定対応（2026-03-25実施）:**
`n_races_valid` / `n_races_test` フィールドをJSONに追加。
少サンプルエントリ（n<30）を目視フラグとして確認可能にした。

**再設計のトリガー:**
2026年実績が n=200 以上蓄積した時点で、
「バックテストROI vs 実績ROI」の乖離を測定し、
乖離が10%pt以上であれば walk-forward再設計を優先する。

## 仮説 H4：G1直後ローテーション馬のRPCI過小評価バイアス（2026-03-25 登録）

**発見の経緯:**
阪神大賞典（2026-03-22）でアドマイヤテラ（1着）に印がつかなかった原因を分析。
前走（有馬記念）のRPCIが52.8（平均ペース）で、この馬の得意条件（RPCI≥56スロー）と乖離。
また前走補9が「--」（欠損）となっており、モデルへの入力が不正確だった。

**仮説内容:**
GIは展開・ペースが特殊（平均ペースが多い）なため、スロー巧者の馬はGIで前走RPCIが
構造的に下がりやすい。モデルはこの「条件不一致による低パフォーマンス」を
「能力低下」と誤読する可能性がある。

**テスト可能な予測:**
「前走GI → 今走G2以下」のローテーションで、前走RPCI<54の馬の複勝率は、
同条件で前走RPCI≥56の馬より統計的に高い（市場が過小評価している）。

**棄却条件:**
n≥200 かつ前走RPCI<54グループのROIが前走RPCI≥56グループのROIより+5%以上高くない場合。

**現状対応（2026-03-25 実施）:**
- 根本修正（前々走データ活用）は架構上困難（kako5は予測時のみ、学習データなし）
- 暫定対応: `predict_weekly.py` に kako5 を使った警告列「警告」を追加
  - 過去5走に中止(止)・除外(外)・取消(消)がある馬に警告表示
  - 例: 「2走前中止・よって未知数」
- 特徴量追加（前走補9欠損フラグ）は別イテレーションで検討

**検証条件:**
n=200以上蓄積後（2026年下半期目処）。
前走レース種別（GI/GII/GIII/OP等）と前走RPCIの交互作用を分析。
