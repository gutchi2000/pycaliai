## Phase G: 馬券構成最適化分析（2026-03-22）

### sim_all_bets.py — 全組み合わせROI検証

**検証期間**: test 4,855レース（2024-2025）
**結果（主要）**:

| 買い目 | ROI | 評価 |
|---|---|---|
| 単勝◎（現行） | 77.3% | ✗ 最低 |
| 単勝○ | 80.8% | ★ |
| **単勝△** | **81.6%** | **★ 最高** |
| 複勝◎ | 81.3% | ★ |
| **複勝○** | **83.8%** | **★ 最高** |
| 馬連◎-▲（単体） | 79.0% | 最良馬連 |
| 馬連◎軸2点（現行） | 77.5% | ✗ |
| 三連複◎○▲ | 64.6% | ✗ 最低 |

**結論**:
- 単勝は◎より△・○の方が ROI が高い（◎は過大評価されている）
- 複勝は○が最も優秀
- 三連複は全組み合わせで ROI < 80%: 廃止を検討すべき
- **現行構成（単勝◎・複勝◎・馬連◎軸2点・三連複◎○▲）は最適でない**

**推奨変更案（検討中）**:
- 単勝◎ → 単勝△ or ○
- 複勝◎ → 複勝○
- 三連複廃止 → 浮いた1点分を他に充当

### sim_ev_select.py — EV選択シミュレーション（調査打ち切り）

**EV = 印の推定確率 × 単勝オッズ** で選択馬を変える試み。

**問題点（2点）**:
1. **kekka_master は上位3着馬のみ収録** → 馬番がヒットする = 上位着馬のみ = 的中率が水増し
   （44,907レース × 3頭 = 134,839行、全馬は含まれない）
2. **BT の `推定的中確率` はレース内全bet行で同値** → 馬ごとの勝率ではない

→ 現行データではEV選択は実装不可。
週次kekka CSV（data/kekka/）は全馬収録だが2026年分のみ（≒700R）、サンプル不足。
**中長期課題: per-horseの単勝オッズを全レース分保存するデータ基盤が必要**。
